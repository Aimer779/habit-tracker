# 习惯打卡 App - 图片效果差距与实现计划

## 目标效果概述

实现一个清新风格的现代化习惯打卡 App，包含：网格卡片布局、习惯图标与主题色、打卡热力图、底部导航 + FAB、筛选排序等功能。

---

## ✅ 当前已实现（底层逻辑可直接复用）

- [x] 添加 / 编辑 / 删除习惯
- [x] 每日打卡 / 取消打卡
- [x] 简单统计页面（总习惯数、今日完成、今日完成率、总打卡次数）
- [x] SharedPreferences + JSON 持久化
- [x] 基础 RecyclerView 列表

## ❌ 与图片效果的差距（按模块分类）

### 1. 整体布局结构

| 图片效果 | 当前实现 |
|---------|---------|
| 底部导航栏（主页 / 统计 / 设置）+ 中间悬浮 FAB | 两个普通按钮"添加习惯" + "统计"在底部 |
| 顶部标题栏（"刻度"）+ 筛选排序栏 | 无标题栏，直接是列表 |
| 2 列网格卡片布局 | 单列列表布局 |

### 2. 习惯卡片样式

| 图片效果 | 当前实现 |
|---------|---------|
| 每个习惯有专属图标（📖 每日阅读、🥚 吃早餐、🏋️ 健身） | 只有文字名称 |
| 卡片内有打卡热力图 / 网格（7×7 圆点矩阵显示历史打卡） | 只有"今日已打卡 / 未打卡"文字 |
| 每个习惯有主题色（黄色、蓝色、红色） | 统一样式，无主题色 |
| 打卡 / 撤销打卡按钮与习惯主题色一致 | 统一的普通按钮 |
| 卡片整体更美观，有圆角、留白、阴影层次 | 较简单的 MaterialCardView |

### 3. 顶部功能栏

| 图片效果 | 当前实现 |
|---------|---------|
| "全部" 下拉筛选 | ❌ 无 |
| 排序按钮 | ❌ 无 |
| 右上角日历 / 网格图标 | ❌ 无 |

### 4. 底部导航

| 图片效果 | 当前实现 |
|---------|---------|
| 主页（✅ 图标） | 用按钮替代 |
| 统计（📊 图标） | 用按钮替代 |
| 设置（⚙️ 图标） | ❌ 无设置页面 |

### 5. 主题与配色

| 图片效果 | 当前实现 |
|---------|---------|
| 整体采用清新浅色主题，配色统一 | 默认 Material 3 主题，无自定义 |
| 薄荷绿 / 青色系主色调 | 无 |

## 🛠️ 需要做的改动（按优先级排序）

- [ ] `Habit` 模型增加 `icon` 字段（图标资源名或 Emoji）

- [ ] `AddHabitActivity` 增加图标选择器和颜色选择器
- [ ] 数据迁移：旧数据兼容处理（新字段默认值）

### Phase 2：习惯卡片重构（核心）
- [ ] 将 `RecyclerView` 改为 `GridLayoutManager`（2 列）
- [ ] 重新设计 `item_habit.xml`：

  - [ ] 添加打卡热力图网格（自定义 View 或 GridLayout）
